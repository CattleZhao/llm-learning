# Project 6: APK 恶意行为分析系统 - 技术方案

> 基于 MCP 和 LLM 的 Android APK 恶意软件检测系统

## 1. 项目概述

### 1.1 项目背景

随着移动应用的普及，恶意 APK 软件对用户安全构成严重威胁。传统的恶意软件检测方法存在以下局限：
- 依赖静态签名库，无法检测新型恶意软件
- 人工分析效率低下，难以应对海量应用
- 检测规则更新滞后

### 1.2 解决方案

本项目采用 **LLM + MCP + 自定义规则库** 的架构，实现：
- 🔍 **自动化分析** - 通过 MCP 调用 JADX 反编译工具
- 📋 **规则引擎** - 支持正则表达式包路径匹配和自定义规则
- 🧠 **智能记忆** - 存储已知恶意软件特征和历史分析结果
- 🤖 **自我反思** - 确保分析完整性

### 1.3 核心价值

| 能力 | 传统方法 | 本系统 |
|------|---------|--------|
| 检测速度 | 人工小时级 | ⚡ 自动分钟级 |
| 新型恶意软件 | ❌ 无法检测 | ✅ LLM 推理检测 |
| 规则更新 | ❌ 手动更新 | ✅ 自动学习 |
| 报告质量 | 简单列表 | ✅ 详细结构化报告 |

## 3. 核心功能

### 3.1 检测维度

| 维度 | 检测方法 | 数据来源 |
|------|----------|---------|
| **权限分析** | 危险权限组合检测 | AndroidManifest.xml |
| **网络通信** | 恶意域名、C2 服务器 | 字符串提取、代码分析 |
| **包路径** | 恶意包路径模式 | 类结构扫描 |
| **API 调用** | 隐私敏感 API 检测 | 代码分析 |
| **代码混淆** | 混淆程度评估 | 类名、方法名分析 |
| **恶意行为** | 已知木马特征 | 恶意软件家族库 |

### 3.2 MCP 工具列表

| 工具名称 | 参数 | 说明 |
|---------|------|------|
| `get_android_manifest()` | 无 | 获取应用清单信息 |
| `get_all_classes(offset, count)` | offset=0, count=0 | 获取所有类列表 |
| `get_class_source(class_name)` | class_name | 获取类的源代码 |
| `get_strings(offset, count)` | offset=0, count=0 | 提取字符串资源 |
| `search_classes_by_keyword()` | 多参数 | 按关键字搜索类 |
| `get_methods_of_class(class_name)` | class_name | 获取类的方法列表 |
| `jadx_open_apk(apk_path)` | apk_path | 在 JADX-GUI 中打开 APK |

### 3.3 自定义规则格式

```json
{
  "rules": [
    {
      "name": "木马_通信包_特征",
      "category": "trojan",
      "severity": "critical",
      "description": "检测木马的典型通信包结构",
      "patterns": [
        "com/malware/trojan/.*",
        ".*\\.payload/.*",
        ".*\\.c2client/.*"
      ],
      "tags": ["trojan", "c2"],
      "references": ["已知木马家族 X"]
    }
  ],
  "metadata": {
    "version": "1.0",
    "last_updated": "2024-03-12"
  }
}
```

## 4. 数据流

### 4.1 分析流程

```
1. 📱 用户上传 APK
   │
   ▼
2. 📄 JADX-GUI 打开 APK (自动)
   │
   ▼
3. 🔌 Agent 连接 MCP Server
   │
   ▼
4. 📋 解析 AndroidManifest.xml
   │  ├─ 提取权限列表
   │  ├─ 检测危险权限组合
   │  └─ 分析组件声明
   │
   ▼
5. 📂 扫描代码结构
   │  ├─ 获取所有类列表
   │  ├─ 匹配恶意包路径
   │  └─ 识别混淆模式
   │
   ▼
6. 🌐 分析网络通信
   │  ├─ 提取 URL/域名
   │  ├─ 检查 C2 服务器
   │  └─ 分析加密通信
   │
   ▼
7. ⚙️ 分析 API 调用
   │  ├─ 检测敏感 API
   │  ├─ 分析反射调用
   │  └─ 识别动态加载
   │
   ▼
8. 📝 提取字符串
   │  ├─ API 端点
   │  ├─ 硬编码密钥
   │  └─ C2 地址
   │
   ▼
9. 🎯 匹配恶意模式
   │  ├─ 应用自定义规则
   │  ├─ 查询知识库
   │  └─ 检索相似历史
   │
   ▼
10. 📊 生成分析报告
    ├─ 计算风险等级
    ├─ 列出发现详情
    └─ 提供缓解建议
```

## 5. Agent 设计

### 5.1 双 Agent 模式

系统支持两种 Agent 模式：

#### 模式 A：硬编码流程 Agent
- 固定的 9 步分析流程
- 确定性输出，易于调试
- 适合标准化分析

#### 模式 B：LLM 驱动 Agent
- LLM 自主决定调用哪些工具
- 更灵活的分析路径
- 支持复杂推理

### 5.2 工具调用示例

```python
# Agent 自动选择工具
response = agent.think(
    "分析 app.apk",
    context={"apk_path": "app.apk"}
)

# LLM 自主调用工具序列
1. jadx_open_apk("app.apk")
2. get_android_manifest()
3. get_all_classes()
4. match_malware_rules(...)
5. 生成报告
```
