# 大模型学习路线设计

**日期**: 2026-03-05
**学习者**: Java后端开发工程师
**目标**: 职业转型到AI/大模型方向
**学习方式**: 项目驱动学习

---

## 一、学习目标

从Java后端开发转型到大模型应用开发方向，通过4个递进项目掌握大模型应用开发的核心技能，积累可展示的项目经验。

---

## 二、学习背景

- **当前背景**: Java后端开发工程师
- **AI基础**: 比入门级强一点但有遗忘
- **可用时间**: 每周10小时以内（业余时间）
- **学习偏好**: 项目驱动，直接从实战入手
- **转型方向**: 应用层开发（RAG、Agent、Prompt Engineering）

---

## 三、整体路径

```
阶段1（2-3周）：基础API调用 → 阶段2（4-6周）：RAG应用 → 阶段3（6-8周）：Agent应用 → 阶段4（3-4周）：多Agent协作
     ↓                            ↓                          ↓                            ↓
  理解大模型                   向量检索+上下文              多步推理+工具                  多Agent对话
  API使用                      增强应用                    自动化决策                      团队协作
```

**总时长**: 4-5个月（每周10小时）
**总投入**: 150-200小时

---

## 四、项目详情

### 项目1：大模型基础应用（2-3周）

**目标**: 掌握大模型API调用，理解Prompt Engineering

**技术选型**:
- 语言: Python（大模型生态成熟）
- 模型: OpenAI GPT-4o / Claude 3.5 / 或国产模型（通义千问、DeepSeek等）

**核心功能**:
1. 基础问答 - 简单的Q&A对话
2. 文档摘要 - 输入长文本，生成摘要
3. 结构化输出 - 让模型输出JSON格式数据

**学习重点**:
- API调用方式
- Prompt设计技巧（角色设定、示例引导、思维链）
- 温度/MaxTokens等参数影响
- Token消耗和成本控制

**产出**: 一个命令行工具，能做问答、摘要和结构化提取

---

### 项目2：RAG知识库问答系统（4-6周）

**目标**: 掌握检索增强生成（RAG），解决大模型知识时效性和准确性问题

**技术选型**:
- 向量数据库: ChromaDB（轻量）或 Milvus（生产级）
- Embedding模型: OpenAI text-embedding-3 / 本地模型（如BGE）
- 框架: LangChain或 LlamaIndex

**核心功能**:
1. 文档处理 - 支持PDF/TXT/MD文档解析和分块
2. 向量化存储 - 文档块转向量并存储
3. 语义检索 - 根据问题检索最相关的内容
4. 上下文增强 - 将检索结果注入Prompt
5. 答案生成 - 基于检索内容回答用户问题

**学习重点**:
- 文档分块策略（固定大小、语义分割）
- 向量Embedding原理和选择
- 相似度计算（余弦相似度等）
- RAG优化技巧（重排序、混合检索、查询改写）

**产出**: 一个Web应用，上传文档后可进行智能问答

---

### 项目3：代码助手Agent（6-8周）

**目标**: 掌握Agent开发，让AI能自主规划和执行多步骤任务

**技术选型**:
- Agent框架: LangChain Agent / AutoGPT / CrewAI
- 工具集成: 搜索、代码执行、文件操作、API调用
- 记忆机制: 短期记忆/长期记忆

**核心功能**:
1. 任务规划 - Agent理解目标并分解步骤
2. 工具调用 - 动态选择和调用外部工具
3. 结果验证 - 检查执行结果，必要时重试
4. 多轮对话 - 保持上下文，支持交互式调整

**应用场景**: 代码助手
- 分析代码结构和逻辑
- 生成单元测试
- 提供重构建议

**学习重点**:
- ReAct框架（推理+行动循环）
- Tool设计规范
- 记忆管理策略
- Agent调试和监控

**产出**: 一个能自主完成代码相关任务的Agent应用

---

### 项目4：多Agent协作团队（3-4周）

**目标**: 掌握多Agent协作框架，理解Agent之间的对话和协作机制

**技术选型**:
- 主框架: AutoGen（微软出品，企业级）
- 对比框架: CrewAI（理解不同设计理念）
- Web框架: Streamlit（可视化界面）

**核心功能**:
1. 角色定义 - 定义不同职责的Agent（编码者、审查者、测试者）
2. 对话协作 - Agent之间通过对话协作完成任务
3. 代码执行 - 在沙箱环境中安全执行生成的代码
4. 结果汇总 - 整合并展示最终成果

**应用场景**: 代码开发团队
- Coder Agent：负责编写代码
- Reviewer Agent：负责代码审查
- Tester Agent：负责测试验证
- UserProxy Agent：负责协调和执行

**学习重点**:
- AutoGen的核心概念（Agent、Conversation、Code Execution）
- System Message设计和角色定义
- Agent对话流程控制
- 对比CrewAI的设计差异

**产出**: 一个多Agent协作的代码开发团队系统，支持命令行和Web两种交互方式
