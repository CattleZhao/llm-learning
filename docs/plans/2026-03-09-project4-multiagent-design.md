## 一、项目概述

### 1.1 学习目标

通过构建一个多 Agent 协作的代码开发团队，掌握：
- AutoGen 框架的核心概念和使用方法
- 多 Agent 之间的协作模式和对话机制
- Agent 角色设计和 System Message 编写技巧
- 对比学习 CrewAI 框架，理解不同框架的设计理念

### 1.2 项目定位

- **难度等级**: 进阶（在 Project 3 单 Agent 基础上升级）
- **学习时长**: 3-4 周（每周 10 小时）
- **核心价值**: 理解多 Agent 协作，掌握 AutoGen 企业级框架

## 三、Agent 角色设计

### 3.1 角色定义

| Agent | 类型 | 角色定位 | 主要职责 |
|-------|------|---------|---------|
| **UserProxy** | UserProxyAgent | 协调者 | 接收用户需求、协调对话、执行代码、汇总结果 |
| **Coder** | AssistantAgent | 编码者 | 理解需求、编写代码、解释实现逻辑 |
| **Reviewer** | AssistantAgent | 审查者 | 检查代码质量、提出改进建议、确保最佳实践 |
| **Tester** | AssistantAgent | 测试者 | 编写测试用例、验证功能、报告测试结果 |

### 3.2 System Message 设计

**UserProxy Agent**:
```python
system_message = """
你是用户代理，负责协调开发团队完成代码任务。
你的职责：
1. 理解用户需求并传达给团队
2. 协调 Coder、Reviewer、Tester 之间的协作
3. 执行生成的代码并验证结果
4. 汇总最终成果并反馈给用户

请用简洁清晰的方式沟通。
"""
```

**Coder Agent**:
```python
system_message = """
你是资深 Python 开发者，负责编写高质量代码。

编码规范：
1. 使用类型注解
2. 编写清晰的函数和变量命名
3. 添加必要的注释和文档字符串
4. 遵循 PEP 8 规范
5. 考虑边界条件和错误处理

输出格式：完整的可执行代码 + 简要说明
"""
```

**Reviewer Agent**:
```python
system_message = """
你是代码审查专家，负责确保代码质量。

审查要点：
1. 代码逻辑正确性
2. 代码规范和风格
3. 潜在的 bug 和边界问题
4. 性能优化建议
5. 安全性问题

反馈格式：具体问题 + 改进建议
"""
```

**Tester Agent**:
```python
system_message = """
你是测试工程师，负责验证代码正确性。

测试策略：
1. 编写全面的测试用例
2. 覆盖正常情况和边界情况
3. 验证错误处理
4. 报告测试结果

测试框架：pytest / unittest
"""
```

## 四、协作流程设计

### 4.1 标准对话流程

```
阶段 1: 需求理解
┌─────────────┐
│ User: 需求  │
└──────┬──────┘
       ▼
┌──────────────────┐
│ UserProxy 分发   │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ Coder 理解需求   │
└──────────────────┘

阶段 2: 代码开发
┌──────────────────┐
│ Coder 编写代码   │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ Reviewer 审查    │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ Coder 修改代码   │ (如需修改)
└──────────────────┘

阶段 3: 测试验证
┌──────────────────┐
│ Tester 编写测试   │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ UserProxy 执行   │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 汇总结果反馈用户 │
└──────────────────┘
```

### 4.2 对话终止条件

- 代码审查通过 + 测试全部通过
- 达到最大轮次限制（防止无限循环）
- 用户手动终止

### 5.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **代码生成** | 根据需求自动生成代码 | P0 |
| **代码审查** | 自动检查代码质量 | P0 |
| **测试生成** | 自动编写和执行测试 | P0 |

| **历史记录** | 保存对话历史供回顾 | P1 |

**阶段 2: Web 界面**

- 展示最终代码和测试结果
